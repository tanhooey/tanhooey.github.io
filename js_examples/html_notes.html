HTML Basics

I. ELEMENTS AND ATTRIBUTES

Elements

HTML is a markup language. It is made up of elements. The typical element contains an opening tag and its attributes, the enclosesd text content, and the closing tag. There are three types of elements:
  • Block element
  • Inline element
  • Void element

Block Elements

The block element is enclosed by two tags, and each element begins with a newline character.

A block element can only be nested inside a block element.

Inline Element

The inline element is also enclosed by two tags, but inline elements do not create a newline when declared.

Inline elements can be nested inside both block and inline elements.

Void Elements

The void element consists of only a single tag, with no content or closing tag. Void tags are usually used to embed something into the page.

Attributes

Attributes are located in the opening tag, and contain information about the element that won't appear in the content. There are three things an attribute should have:

  • a space between it and the element name.
  • the attribute name, followed by the equal sign.
  • an attribute value, wrapped in quotations.

Attributes can be written without values. These attributes are called boolean attributes. Boolean attributes can only have one value, and its generally the same as the attribute name.

For example, the boolean attribute "disabled" requires you to only write "disabled" in the tag.

Regarding attribute values, they may be wrapped in either single or double quotes. Things to keep in mind:

  • If you are using single quotes and the value contains single quotes as well (in the case of apostrophes), you must use HTML entities.
  • Do not mix double and single quotes, or else you will get unexpected behavior.
  • While you technically can omit quotes when specifying values, it's best practice to keep them in to avoid unspecified behavior.

II. ANATOMY OF AN HTML DOCUMENT

The Skeleton

There are several key elements that make up an HTML document. They include:

  • <!DOCTYPE html> Include this at the top of all your HTML docs
  • <html></html> Besides DOCTYPE element, wrap all other elements in doc within this html tag
  • <head></head> Acts as a container for everything you want to include on the HTML page, that isn't the content the page will show to viewers. This includes keywords and descriptions that would appear in search results, CSS style stuff, and others.
    • <meta charset="utf-8"> This is the meta tag. It represents metadata that cannot be represented by other HTML meta-related elements. The charset attribute allows usage of characters found in most languages.
    • <title></title> Sets the title for the page, displayed on the tab.
  • <body></body> Wraps all other content on the page, and is itself wrapped by the html element.

III. THE HEAD

What's in the Head?

  • The title (what appears on the tab/bookmark by default)
  • Metadata, including:
    • charset
    • name, which specifies the type of meta element.
    • content, which describes the actual meta content.

CSS and JavaScript

The <link> and <script> elements are the most common CSS and JavaScript elements found within the head, respectively.

  • The <link> element has a rel and href attribute. The rel attribute provides the name of the CSS stylesheet used, and href provides the path fo the stylesheet.
  • The <script> element has a src attribute that points to the JavaScript to be loaded in. The boolean attribute defer allows for all HTML elements to be loaded in before the JavaScript starts loading.
    • The script element is not a void element, so remember the opening and closing tags.
    • Instead of pointing to an external script file using src, you can also put it inside the script element.

IV. HTML TEXT FUNDAMENTALS

What are best practices when it comes to using headings in HTML?

  • Use only one <h1> tag per page.
  • Make sure headings sit in correct order according to hierarchy.
  • Of the six headings available, aim to use no more than three.

What are the advantages of giving a page structure?

Giving the page structure allows it to be indexed more easily by web engines and users alike. It also helps the visually impaired by allowing text-to-speech to operate more accurately. Lastly, it allows us to style content specifically with CSS.

In the case that we want to change how an element appears visually but don't want to modify its semantics, we can use the <span> element.

Lists and Emphases

There are two types of list in HTML: unordered, denoted by <ul> , and ordered, denoted by <ol>. The elements of each list are created by nesting <li> list elements within the list type.

There are different inline elements used to style text:

Element Description
<strong></strong> AND <b></b> Bolds content.
<em></em> AND <i></i> Italicizes content.
<mark></mark> AND <u></u> Highlights/underlines content.

The strong, em, and mark elements have semantics associated with them, while b, i, and u do not. MDN recommends avoiding b, i, and n, and instead consider using CSS styling.


V. Hyperlinks

Anatomy of a Hyperlink

Hyperlinks are inline elements that wrap the content that you want to convert into a hyperlink. This includes both inline (images and text) and block level (paragraphs and headings) elements.

The link element includes opening and closing tags <a></a>. Inside the opening tag there is the href attribute, which provides the path to the web material.

The link element may also contain a title attribute, which provides additional information when users hover over the hyperlink.

URLs and Paths

URLs use paths to find files. Paths are organized in a directory structure, with the main directory called the root and subsequent directories beneath the root being called subdirectories.

In a website, the index.html file in the root directory is the landing page for the website. Other index.html files can exist, but they must be found in other directories. Numerous index.html files can be landing pages for other parts of the website.

To access subdirectories, use the / character. To go higher up the directory hierarchy, use the .. characters.

In HTML, it's possible to link to just a specific part of an HTML document, or document fragment. This involves adding an id attribute to the block element you want to go to. Then, at the end of the URL, you would add a pound sign (#) and the id name. This can be done to jump to a specific part of the current page as well.

    Example:
  • In the element opening tag: <h1 id="this_head">The content</a>
  • In the link itself: <a href="url#id"/>
