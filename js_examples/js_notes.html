VI. JavaScript

Basic Overview

Javascript is a light-weight, interpreted programming language. The functionality of JavaScript can be divided into two classes: client-side and APIs.

Client-side functionalities include:

  • Storing information into variables
  • Performing operations on strings and other content on page
  • Running code in response to certain events on a page (i.e. a click)

Application Programming Interfaces, or APIs, are ready made sets of code that allow programmers to implement certain functionalities that would be hard/impossible to build by scratch. APIs fall into two categories:

  • Browser APIs are built into the browser you're using. They are able to expose data regarding the surrounding computer environment and perform other complex tasks. An important browser API is the DOM (Document Object Model) API, which allows for dynamic changing of HTML and CSS on the webpage.
  • Third-party APIs are not built into the browser by default. An example is the Twitter API.

Client-side vs Server-side Code, Dynamic vs Static

Client-side code is code that is run on the user's machine.

Server-side code is code that is run on the server. The results of the code are downloaded and displayed in the browser.

Dynamic code is code that can update a webpage based on different circumstances. This can happen both on the server and client side.

Static code is code that does not change content of a webpage at all.

Applying JS to Your Site

JavaScript is integrated into HTML using the <script> element (remember it is not a void element; a closing tag is needed). There are multiple ways this can be used:

  1. Internal JavaScript: The JavaScript code is found right in our HTML. It is inserted as the content of the <script> element in the head of the document.
  2. External JavaScript: The JavaScript code is imported from an external .js file. The path to the file is specified via the src tag within the <script> element. The content of the script element is empty.

HTML/CSS First, JS Last

Many issues can arise if you try to load JS simultaneously/before the HTML is completely loaded onto the page.

Variables and Constants

Variables are names for values. You create a variable with the keyword let, followed by the name of your variable.

Constants are also used to name values, but they can't be changed once they've been set. You create a constant with the keyword const.
